# dre-droidspaces-recovery — docs

Slot-B recovery-style boot for the **OnePlus Nord N200 5G** (`dre`)
that turns the device into a Linux phone: custom kernel + Android-
recovery-style ramdisk + Ubuntu container running under Droidspaces +
recovery-console as the on-panel UI with an on-screen keyboard.

## Sibling repos

| repo | role | local path |
|---|---|---|
| [dre-droidspaces-kernel](https://github.com/alltechdev/dre-droidspaces-kernel) | reproducible kernel `Image` (Droidspaces patches + touch force-resume + WCN3990 wifi-in-recovery hooks) | `~/dre` |
| **dre-droidspaces-recovery** (this) | builds slot-B `boot.img` + `vendor_boot.img`, init.rc, scripts | `~/fromrecovery` |
| [recovery-console (fork)](https://github.com/alltechdev/recovery-console) | the binary launched on the panel as the UI | `~/recovery-console` |
| [cnss-stage](https://github.com/alltechdev/cnss-stage) | userspace WLAN orchestrator staging (`libperipheral_client` shim + cnss-daemon staging area) | `~/cnss-stage` |

## Boot-time flow (slot B)

```
bootloader
  └─ kernel boots from boot_b
      └─ init reads vendor_boot_b ramdisk's init.rc
          ├─ on init        → start selinux-permissive (sets enforce=0)
          ├─ on boot        → start lineage recovery binary (powers up DRM/backlight)
          │                   start recovery-console-bringup (oneshot, sleep 2)
          │                       └─ start recovery-console
          │                           └─ /system/bin/recovery-console --exec /system/bin/boot-ubuntu.sh
          │                               ├─ stop recovery (kills lineage recovery, takes DRM)
          │                               ├─ pokes /proc/touchpanel/force_resume to wake the touch IC
          │                               ├─ initializes DRM via legacy SETCRTC
          │                               ├─ paints the term grid + on-screen keyboard
          │                               └─ exec's boot-ubuntu.sh inside its PTY
          │                                   ├─ mounts /linux (ext4 on /dev/block/by-name/userdata)
          │                                   └─ exec's droidspaces with -r /linux/rootfs --foreground
          │                                       └─ Ubuntu systemd boots inside the container
          │                                           └─ /bin/sh on tty1 (visible on the panel)
          ├─ on boot        → start wifi-bringup (sleep 6; /linux/wifi-bringup.sh)
          │                       (force_powerup → wlan_pd → icnss → cnss-daemon → wlan.ko → wlan0)
          │                       (then auto-reconnect via /linux/wpa.conf if present)
          ├─ on boot        → start powerkeyd (/linux/powerkeyd.sh)
          │                       (holds wake_lock; on bl_power 4->0 wake transitions, fires
          │                        /proc/touchpanel/force_resume; brightness keeper at 80%)
          └─ adb + USB NCM ethernet (gadget configfs setup)
```

See [INIT-FLOW.md](INIT-FLOW.md) for the gory init.rc walk,
[WIFI-BRINGUP.md](WIFI-BRINGUP.md) for the WLAN chain,
[POWER-AND-WAKE.md](POWER-AND-WAKE.md) for the power-button + touch-on-wake handling,
[GPU.md](GPU.md) for hardware-accelerated EGL/GLES on Adreno 619.

## Build pipeline

`./build.sh` reads the **stock** dre `boot.img` and `vendor_boot.img`
from `~/dre/`, swaps in the kernel from
`~/dre/out/arch/arm64/boot/Image` (or the convenience copy at
`~/fromrecovery/kernel-Image-droidspaces`), overlays our ramdisk on
top of the stock ramdisk, and repacks both.

Critical detail: `build.sh` chmods the overlay files in the work tree
to `0644` (regular) / `0755` (system/bin) before `cpio -o`. **Do not
remove this step.** Files edited under the host's default umask 0002
land at 0664 / 0775; cpio carries those mode bits straight into the
archive; Android init silently refuses to load init.rc if it's group-
writable and the device hangs at the OEM splash with no log output.
This bricked us once for an hour during phase 6. See
[KNOWN-ISSUES.md](KNOWN-ISSUES.md).

## What gets flashed

After `./build.sh`:

```
fastboot flash vendor_boot_b out/vendor_boot-slot-b.img
fastboot flash boot_b out/boot-slot-b.img
fastboot reboot recovery
```

Always `fastboot reboot recovery` — bare `fastboot reboot` boots slot
B in *normal* mode which has no Android user data and just hangs.
[memory: feedback_recovery_reboot.md]

## Userdata layout

We `mkfs.ext4`'d `/dev/block/by-name/userdata` and use it as `/linux`
on the device:

```
/linux/
├── bin/droidspaces           # 324K static aarch64 binary, v6.0.0
├── rootfs/                   # Ubuntu 24.04 XFCE base + phosh apt-installed
│                             # ravindu644/Droidspaces-rootfs-builder v20260419
├── modules/wlan.ko           # vendor_dlkm wlan.ko, currently unused
├── firmware-vendor/          # extracted vendor firmware (ipa_fws etc)
└── container.config          # auto-generated by droidspaces
```

Android slot A still flashes/boots, just with no user data, since
this filesystem replaced the encrypted Android `/data`.
